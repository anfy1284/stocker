# Архитектура проекта Stocker

Технические детали реализации (классы, модули, схемы, цепочки вызовов).
Обновляется по мере роста проекта. Общие принципы работы — в
[`ИНСТРУКЦИИ_ДЛЯ_AI.md`](ИНСТРУКЦИИ_ДЛЯ_AI.md), спецификация — в [`ТЗ.md`](ТЗ.md).

## Общая форма

Модульный конвейер поверх одной SQLite-БД и файлового хранилища превью. Модули
запускаются независимо и обмениваются данными только через БД. Единая точка
запуска — CLI `python -m stocker <команда>`.

## Структура кода

```
stocker/
  __init__.py         версия пакета
  __main__.py         точка входа `python -m stocker`
  cli.py              разбор команд (argparse), диспетчеризация
  config.py           загрузка конфигурации из окружения/.env
  logging_setup.py    настройка логов (консоль + ротируемый файл)
  db.py               соединение с SQLite, схема и миграции
  intake.py           приём (M1): скан входящих, дедуп, EXIF, превью
```

Обвязка в корне: `requirements.txt`, `.env.example`, `.gitignore`, `README.md`,
документы `ТЗ.md` / `ПЛАН_РЕАЛИЗАЦИИ.md` / этот файл.

## Конфигурация (`config.py`)

- `PROJECT_ROOT` — корень проекта (родитель пакета).
- `load_config()` — вызывает `load_dotenv(.env)`, затем собирает `Config` из
  переменных окружения. Все пути перекрываются переменными `STOCKER_*`; по
  умолчанию всё в `<PROJECT_ROOT>/data` (`inbox`, `previews`, `logs`,
  `stocker.db`).
- Секрет `ANTHROPIC_API_KEY` — только из окружения, в коде и логах не хранится
  (`as_display_dict()` отдаёт `<задан>/<не задан>`).
- `Config.ensure_dirs()` — идемпотентное создание всех рабочих каталогов.

## Логирование (`logging_setup.py`)

`setup_logging(logs_dir, level)` — идемпотентная настройка корневого логгера:
консоль + `RotatingFileHandler` на `logs/stocker.log` (1 МБ × 3). Флаг
`_stocker_configured` на корневом логгере защищает от повторного добавления
обработчиков.

## База данных (`db.py`)

- `get_connection(db_path)` — соединение с `row_factory=Row` и
  `PRAGMA foreign_keys = ON`.
- Миграции: `_MIGRATIONS` — список функций «поднять схему на 1 версию»;
  индекс+1 = целевая версия. `SCHEMA_VERSION = len(_MIGRATIONS)`.
  `init_db(db_path)` создаёт файл при первом запуске и применяет недостающие
  миграции по `PRAGMA user_version`. Каждый шаг проекта дописывает свою
  миграцию — схема растёт предсказуемо.
- **v1 (Шаг 2):** таблица `assets` — поля приёма (`original_path`,
  `preview_path`, `content_hash` UNIQUE, `file_type`, `file_size`,
  `width`/`height`, `captured_at`, `camera_make`/`camera_model`,
  `orientation`, `status`, `created_at`) + индекс по `status`. Поля
  классификации/групп/метаданных/загрузки добавят свои шаги.
- Статусы конвейера — константы (`STATUS_NEW = "new"`).

## Приём (`intake.py`, M1)

`run_intake(cfg)` сканирует `inbox_dir` (рекурсивно, JPEG), для каждого файла:
SHA-256 содержимого → дедуп по `content_hash` (проверка + UNIQUE в БД) → EXIF
(`_read_exif`: дата съёмки `DateTimeOriginal`, `Make`/`Model`, `Orientation`)
→ нормализованное превью (`_make_preview`: `exif_transpose` применяет
ориентацию к пикселям, `thumbnail` до 1600px по длинной стороне, JPEG q85,
имя = `<hash>.jpg`) → вставка записи со статусом `new`. `width`/`height` —
размеры в ориентации показа. Возвращает статистику
(`scanned/added/duplicate/errors`); ошибки на файл логируются и не валят прогон.
Поддержка форматов сейчас: только JPEG (RAW/HEIC — Шаг 6).

## CLI (`cli.py`)

- Команды: `init` (по умолчанию) — `ensure_dirs()` + `init_db()`; `config` —
  печать конфигурации; `intake` — `ensure_dirs()` + `init_db()` + `run_intake()`.
- `main()`: `_force_utf8_streams()` (кириллица в консоли Windows) → разбор
  аргументов → `load_config()` → `setup_logging()` → выполнение команды.

## Журнал шагов

- **Шаг 1 (готово):** каркас, конфигурация, логирование, пустая БД. Задел под
  миграции через `PRAGMA user_version`.
- **Шаг 2 (готово):** приём JPEG. Миграция v1 (`assets`), модуль `intake.py`,
  команда `stocker intake`. Дедуп по SHA-256, EXIF, превью 1600px/q85. Проверено
  на сгенерированных JPEG: записи с EXIF заводятся, превью ужимаются до 1600 по
  длинной стороне, повторный файл не заводится дважды.
