## Общая форма

Модульный конвейер поверх одной SQLite-БД и файлового хранилища превью. Модули
запускаются независимо и обмениваются данными только через БД. Единая точка
запуска — CLI `python -m stocker <команда>`.

## Структура кода

```
stocker/
  __init__.py         версия пакета
  __main__.py         точка входа `python -m stocker`
  cli.py              разбор команд (argparse), диспетчеризация
  config.py           загрузка конфигурации из окружения/.env
  logging_setup.py    настройка логов (консоль + ротируемый файл)
  db.py               соединение с SQLite и инициализация схемы
```

Обвязка в корне: `requirements.txt`, `.env.example`, `.gitignore`, `README.md`,
документы `ТЗ.md` / `ПЛАН_РЕАЛИЗАЦИИ.md` / этот файл.

## Конфигурация (`config.py`)

- `PROJECT_ROOT` — корень проекта (родитель пакета).
- `load_config()` — вызывает `load_dotenv(.env)`, затем собирает `Config` из
  переменных окружения. Все пути перекрываются переменными `STOCKER_*`; по
  умолчанию всё в `<PROJECT_ROOT>/data` (`inbox`, `previews`, `logs`,
  `stocker.db`).
- Секрет `ANTHROPIC_API_KEY` — только из окружения, в коде и логах не хранится
  (`as_display_dict()` отдаёт `<задан>/<не задан>`).
- `Config.ensure_dirs()` — идемпотентное создание всех рабочих каталогов.

## Логирование (`logging_setup.py`)

`setup_logging(logs_dir, level)` — идемпотентная настройка корневого логгера:
консоль + `RotatingFileHandler` на `logs/stocker.log` (1 МБ × 3). Флаг
`_stocker_configured` на корневом логгере защищает от повторного добавления
обработчиков.

## База данных (`db.py`)

- `get_connection(db_path)` — соединение с `row_factory=Row` и
  `PRAGMA foreign_keys = ON`.
- `init_db(db_path)` — создаёт файл при первом запуске, возвращает признак
  создания. Версия схемы — в `PRAGMA user_version`; `SCHEMA_VERSION` (сейчас 0)
  поднимается каждым шагом, добавляющим таблицы. Задел под пошаговые миграции.

## CLI (`cli.py`)

- Команды: `init` (по умолчанию) — `ensure_dirs()` + `init_db()`; `config` —
  печать конфигурации.
- `main()`: разбор аргументов → `load_config()` → `setup_logging()` →
  выполнение команды. Возвращает код выхода.
